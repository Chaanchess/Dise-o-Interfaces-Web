Animaciones

Guía de animaciones en CSS3: Con animaciones podemos desplazar y/o cambiar de color un elemento en un espacio de tiempo.

animation-name

Toda animación en CSS3 consta de 2 propiedades básicas, sin éstas, no podría existir una animación. La primera de ellas es animation-name, esta propiedad asocia el elemento que se desea animar con la regla específica que desencadenará el proceso de animación. Veamos un ejemplo: 

                    #ejemplo {
                        width: 100px;
                        height: 100px;
                        background-color: orange;
                        animation-name: ej_anim;
                        animation-duration: 15s;
                    }
                    
                    @keyframes ej_anim {
                        [...]
                    }

El fragmento de código anterior es un ejemplo de cómo se hace una "llamada" a la regla de una animación. Si se observa en la regla que hace referencia al id #ejemplo, donde se definen las características del objeto a animar (color, forma, tamaño, etc) del que partirá la animación, vemos también en negrita la propiedad animation-name que, como valor, posee el nombre de la regla que desencadenará la animación.

Posteriormente, se puede apreciar una segunda regla llamada "@keyframes ej_anim", esta regla será la encargada de desencadenar la propia animación y, en su interior, contiene una serie de propiedades y valores que determinarán la posición y/o el color a lo largo del tiempo, entre otras cosas que se verán a continuación, del elemento a animar. En este caso, el elemento a animar se trataría del elemento con el id #ejemplo.

@keyframes

Antes de ver la segunda propiedad esencial para ejecutar una animación, vamos a ver la regla @keyframes y como se pueden definir propiedades de formas diferentes. Veamos un par de ejemplos:

Ejemplo 1: from...to.

                    @keyframes ej_anim {
                        from  {background-color:red; left: 0px;}
                        to {background-color:blueviolet; left: 400px;}
                    }

La primera forma con la que se pueden definir propiedades en la regla @keyframes es utilizando las palabras "from" y "to". Como su propio significado indica, la animación comenzará desde un estado de partida "from" y concluirá en un estado de finalización "to", es decir, que adquiere diferentes valores desde un punto inicial hasta uno final. En este ejemplo la animación variará su color del rojo con una posición de separación con el margen izquierdo de 0px hasta el color violeta con una separación de 400px.

Ejemplo 2: porcentajes.

                    @keyframes ej_anim {
                        10%  {background-color:orange; left:0px; top:0px;}
                        25%  {background-color:yellow; left:200px; top:0px;}
                        50%  {background-color:green; left:400px; top:0px;}
                        75%  {background-color:blue; left:200px; top:0px;}
                        100% {background-color:blueviolet; left:0px; top:0px;}
                    }

La otra forma con la que se puede definir las propiedades que ejecutará la animación es con porcentajes. Como se puede apreciar en el código de arriba, los porcentajes designan en qué momento del tiempo que durará la animación se alcanzará la siguiente modificación. Siguiendo el ejemplo de arriba, cuando la animación se haya ejecutado en un 25% de su duración total, habrá alcanzado el color amarillo y una distancia del margen izquierdo de 200px.

animation-duration

La segunda propiedad imprescindible en toda animación es animation-duration, esta propiedad determina cuántos segundos durará dicha animación. Veamos como ejemplo el siguiente código:

                    #luzroja {
                        width: 100px;
                        height: 100px;
                        background-color: rgb(255, 0, 0);
                        border-radius: 100%;
                        animation-name: luz_roja;
                        animation-duration: 15s;
                        animation-iteration-count: infinite;
                    }
                    
                    @keyframes luz_roja {
                        15% {background-color: rgb(255, 0, 0);} 
                        20% {background-color: rgba(194, 0, 0, 0.829);}
                        60% {background-color: rgba(194, 0, 0, 0.829);}
                        65% {background-color: rgb(255, 0, 0);}
                        100% {background-color: rgb(255, 0, 0);}
                    }

Como se puede apreciar en el código de arriba tenemos la regla que hacen referencia al id #luzroja y, dentro de ésta, la propiedad animation-duration. Dicha propiedad determina la duración de la animación, en este caso, será de 15 segundos.

Una vez hemos fijado el tiempo de la animación (15s) podemos determinar cuando se realizarán cambios dentro de la misma con porcentajes. De forma que, usando el ejemplo de arriba, podemos apreciar que la luz roja parte de un estado inical en la regla #luzroja usando como color de fondo "rgb(255, 0, 0)", mientras que, dentro de la regla "@keyframes luz_roja" determinamos con porcentajes en qué momento de esos 15 segundos se van a producir cambios en la animación (en este ejemplo, los cambios solo serán de variación de color).

Ejemplo 2

                    #animation-duration-10
                        {
                        width: 100px;
                        height: 50px;
                        background: rgb(209, 86, 86);
                        position: relative;
                        animation: animation-duration infinite;
                        animation-duration: 10s;
                        }
                    @keyframes animation-duration {
                        from {left: 0px;}
                        to {left: 800px;}
                        }

animation-delay

Especifica el tiempo de retardo que debe transcurrir antes de comenzar la animación.
Esto permite que la secuencia de animación no comience nada más ser aplicada a un elemento.
Su valor por defecto es 0, a menos que indiquemos alguno en segundos o milisegundos.
Ej.: 20s --> Segundos
      20ms --> Milisegundos

                    #animation-delay-15 {
                        width: 140px;
                        height: 50px;
                        background: rgb(137, 96, 156);
                        position: relative;
                        animation: animation-delay 10s infinite;
                        animation-delay: 2s;
                        }

animation-iteration-count

Esta propiedad, como se puede deducir de su nombre, añade la característica de repetir una animación ya creada n veces. Para este ejemplo, vamos a repetirla 3 veces. Aquí un ejemplo resaltando la nueva propiedad en negrita: 

                    #anim3 {
                        width: 100px;
                        height: 100px;
                        border-radius: 100%;
                        position: relative;
                        background-color: red;
                        text-align: center;
                        animation-name: animation2_3;
                        animation-duration: 4s;
                        animation-iteration-count: 3;
                    }
                    
                    @keyframes animation2_3 {
                        10%  {background-color:orange; left:0px; top:0px;}
                        25%  {background-color:yellow; left:200px; top:0px;}
                        50%  {background-color:green; left:400px; top:0px;}
                        75%  {background-color:blue; left:200px; top:0px;}
                        100% {background-color:blueviolet; left:0px; top:0px;}
                    }

Como dato de interés, podemos repetir de forma indefinida una animación asignando el valor infinite a la propiedad animation-iteration-count.

animation-direction

Esta propiedad determina la dirección o sentido con el que se va a reproducir la animación. Es una propiedad especial, ya que su comportamiento variará en función del valor que se le asigne. Es una propiedad a la que se le puede asignar cuatro valores, a saber:

  • normal: Con el valor normal la animación realiza el recorrido asignado, es decir, se realizará la animación por defecto como si no se hubiese empleado esta propiedad.
  • reverse: Si usamos el valor reverse la animación se ejecutará en sentido opuesto al señalado. En el caso anterior el cuadrado rotaba en sentido de las agujas del reloj pero, en este, rota en sentido opuesto.
  • alternate: El valor alternate ejecuta la animación tanto en sentido normal como inverso. Primeramente ejecutará la versión en el sentido normal y, posteriormente, la versión invertida. Para ello es imprescindible emplear la propiedad anterior animation-iteration-countpara que haga, como mínimo, las dos vueltas.
  • alternate-reverse: Finalmente alternate-reverse que, como se puede deducir, es un alternate que, en vez de empezar la ejecución predefinida de la animación empieza por el sentido inverso y luego ejecuta el sentido normal. De igual modo, necesitaremos hacer uso de animation-iteration-count para hacer como mínimo las dos vueltas.

                        height: 100px;

                        background-color: red;
                        animation-name: animation4;

animation-timing-function

Esta propiedad describe la curva de velocidad que tomará la animación, es decir, que en vez de mantener una velocidad constante, se puede establecer que en determinados momentos acelere o decelere según esté prefijado para cada valor de la propiedad. Éstos son los valores que puede tomar:

  • linear: Con el valor linear la animación adquiere la velocidad por defecto, como si no se emplease esta propiedad.
  • ease: Con el valor ease la animación arranca con un inicio ligeramente lento que se acelera progresivamente y, conforme se aproxima al final, decelera nuevamente.
  • ease-in: Con ease-in la animación comienza con un inicio muy lento y, cercano al final, incrementa al máximo la velocidad.
  • ease-out: Al emplear ease-out la animación comienza a la máxima velocidad y, cercano al final, la disminuye al mínimo.
  • ease-in-out: Finalmente, con ease-in-out la animación comienza muy lentamente, acelera al máximo, alcanzando su máxima velocidad a medio recorrido, y de nuevo disminuye la velocidad al mínimo aproximándose al final.
  • step-start: steps (valor, start); Comienza antes del intervalo.
  • step-end: steps (valor, end); Comienza después del intervalo. Si el 2º parámetro se omite es end
  • cubic-bezier(P0, P1, P2, P3): Define una curva de bezier. Con cubic-bezier podemos representar los demás valores anteriores. Ej: linear = cubic-bezier(0, 0, 1, 1);

Ejemplo:

                    #animation-timing-function-cubic-bezier {
                        width: 120px;
                        height: 50px;
                        background: rgb(121, 223, 126);
                        position: relative;
                        animation: animation-timing-function 10s infinite;
                        animation-timing-function: cubic-bezier(0, 0, 1, 1);
                    }

                    @keyframes animation-timing-function {
                        from {left: 0px;}
                        to {left: 600px;}
                    }

animation-fill-mode

Una animación jamás afecta al elemento antes del primer fotograma o después del último. La propiedad animation-fill-mode permite anular este comportamiento.

La propiedad animation-fill-mode posee 4 valores, a saber:

  • none: Si empleamos el valor none la animación no aplicará ningún cambio, como si no se emplease esta propiedad.
  • forwards: Al utilizar el valor forwards el elemento conservará los valores del último fotograma.
  • backwards: Con el valor backwards el elemento adquirirá el valor del primer fotograma de la animación y lo conservará durante todo el tiempo de delay que pueda tener la misma, acabada su animación volverá a su estado normal.
  • both: Al emplear both, el elemento adquirirá el valor del primer fotograma de la animación, conservándolo durante todo el tiempo de delay que pueda tener la misma, y acabada su animación continuará con los valores del último fotograma.

En este caso, para comprender el concepto de esta propeidad, no he podido poner en bucle las animaciones de este apartado, por tanto, si has llegado hasta aquí y las animaciones del ejemplo están detenidas te invito a refrescar la página para poder visualizarlas.

animation-play-state

La propiedad animation-play-state determina si una animación está pausada o en funcionamiento. Puede poseer 4 valores diferentes que son: paused, running, initial, inherit.

  • paused: Como su propio nombre indica, al emplear paused la animación permanecerá pausada.
  • running: Si empleamos el valor running la animación pasará a funcionar normalmente. Tanto este valor, como el anterior, es frecuentemente usado por scripts de JavaScripts para manipular el proceso de una animación.
  • initial: Con initial la animación funcionará por defecto como si no empleasemos esta propiedad.
  • inherit: Al usar inherit la animación heredará las propiedades de su elemento padre.

combinación de propiedades en una única línea

Para acabar, existe una forma de definir una animación inluyendo en una única línea todas sus propiedades. Para ello, se debe seguir un orden específico, veamos cómo:

                        #anim8 {
                            width: 100px;
                            height: 100px;
                            border-radius: 100%;
                            position: relative;
                            background-color: red;
                            text-align: center;
                            animation: animation8 5s linear 1s infinite alternate running;
                        }

Usando el código de arriba, la línea maracada en negrita es la forma con la que podemos declarar todas las propiedades de la animación en una línea. Para ello empleamos la propiedad animation seguido de las siguientes propiedades en orden:

  • animation8: Nombre que referencia a la regla @keyframe.
  • 5s: Duración de la animación.
  • linear: Curva de velocidad que posee la animación.
  • 1s: Delay de la animación.
  • infinite: Número de iteraciones.
  • alternate: Dirección de la animación.
  • running: Estado de la animación pausada o en marcha.
